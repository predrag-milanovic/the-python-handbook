# Functions

This chapter teaches how to define and use functions — inputs, returns, and basic patterns for organizing code.

## Lessons

1. [01 - Functions](./01-functions.md) - Learn what functions are and why they help you reuse and organize code.
2. [02 - Function Review](./02-function-review.md) - Step through a function call and understand execution order and returns.
3. [03 - Multiple Parameters](./03-multiple-parameters.md) - Use functions that accept more than one input and why order matters.
4. [04 - Printing vs. Returning](./04-printing-vs.-returning.md) - Understand the difference between printing and returning values.
5. [05 - Where to Declare Functions](./05-where-to-declare-functions.md) - Learn where to define functions so they are available when needed.
6. [06 - Order of Functions](./06-order-of-functions.md) - Organize helpers and entry points to keep runtime order predictable.
7. [07 - None Return](./07-none-return.md) - See what happens when a function doesn't explicitly return a value.
8. [08 - Multiple Return Values](./08-multiple-return-values.md) - Return and unpack multiple values from a function.
9. [09 - Parameters vs. Arguments](./09-parameters-vs.-arguments.md) - Distinguish between parameter names and call-time argument values.
10. [10 - Default Values](./10-default-values.md) - Make parameters optional by giving them default values.
