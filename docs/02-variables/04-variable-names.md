# Variable Names

Variable names must not contain spaces. They are continuous strings of characters.

Python style favors `snake_case` for variable names. [Guido van Rossum](https://en.wikipedia.org/wiki/Guido_van_Rossum), the creator of Python, has encouraged this style, and it keeps code consistent and easy to read.

Here are a few common naming styles:

| Name | Description | Code | Language(s) that recommend it |
| --- | --- | --- | --- |
| Snake Case | All words are lowercase and separated by underscores | `num_new_users` | Python, Ruby, Rust |
| Camel Case | Capitalize the first letter of each word except the first one | `numNewUsers` | JavaScript, Java |
| Pascal Case | Capitalize the first letter of each word | `NumNewUsers` | C#, C++ |
| No Casing | All lowercase with no separation | `numnewusers` | ...just don't do this |

Your Python code will still work with camel case or Pascal case, but `snake_case` is the style we want to use in this handbook.
